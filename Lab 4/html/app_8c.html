static

Contains functions to open and initialize peripherals.

Function to open a PWM signal out of LETIMER0 at a particular frequency duty cycle. This includes values/settings for our specific application.

This part of the code set up the peripheral of the letimer pwm function while it uses the cmu clock to show the letimer clock tree while generating and routing the GPIO pins and the LETIMER function to be able to generate the PWM signal. This sets the period and the active period for which the PWM signal it sent.

Note

This routine defines the values of the APP_LETIMER_PWM_TypeDef struct specific to our application then passes it to the lower level driver letimer_pwm_open function along with which LETIMER we want to use (there is only one LETIMER on the Pearl Gecko, so it will always be 0).

Note
This function is used to setup PWM for this app.
Parameters
[in]periodPWM period in seconds
[in]act_periodPWM active period in seconds

void app_peripheral_setup ( void  )

This function would be used to initialize all peripherals for this application.

This part of the function uses the CMU clock to enable the LETIMER clock tree, and initializes the GPIO and LETIMER needed to produce a PWM signal. This function also adds a call to the si7021 i2c function.

Note
This function only needs to be called only once at the beginning of main function.

void scheduled_letimer0_uf_cb ( void  )

Handles the letimer0 underflow event.

This function clears the scheduled event and then handles the underflow event.

Note
Every time that we enter the LETIMER0 UF event service routine, we will cycle through the energy modes defining what the lowest energy mode that the sleep routine will enter. With the change of the temoerature sensor we no longer needed the the energy mode to be incremented from EM0 to EM4 as the energy mode required for the temperature sensor was set to EM2 in the i2c.c file.
